Istio-Envoy Objects Relationship

Review Red Hat Service Mesh objects

When a new Service Mesh object is created in a specific namespace, a set of configurations blocks are added to Envoy proxies in order to be able to apply these rules in the mesh traffic flow.

In this step, you will review how these objects affect Envoy proxies getting their configuration from Envoy itself.

Please, replace <user_namespace> and <pod_id> with the values provided by the Instructor at the beginning of this tutorial or values obtained from openshift cluster.

Destination Rules

Before applying changes, please save the original Envoy proxy configuration following next steps:

oc exec back-golang-v1-<pod_id> -c istio-proxy -n <user_namespace> curl localhost:15000/config_dump > back-golang-v1-<pod_id>-destination-rule-orig.json
{
 "configs": [
  {
   "@type": "type.googleapis.com/envoy.admin.v3.BootstrapConfigDump",
   "bootstrap": {
    "node": {
     "id": "sidecar~10.128.2.23~back-golang-v1-<pod_id>.<user_namespace>~<user_namespace>.svc.cluster.local",
...

It is time to modify a Destination Rule in order to see how it is implemented in Envoy proxy configuration through the respective configuration blocks:

oc process -f 04-istio-envoy-relationship/00-istio-envoy-rel-drs.yaml --param-file=params.env --ignore-unknown-parameters | oc apply -f -

Once this new Destination Rule has been modified, save the custom Envoy proxy configuration following next steps:

oc exec back-golang-v1-<pod_id> -c istio-proxy -n <user_namespace> curl localhost:15000/config_dump > back-golang-v1-<pod_id>-destination-rule-custom.json
{
 "configs": [
  {
   "@type": "type.googleapis.com/envoy.admin.v3.BootstrapConfigDump",
   "bootstrap": {
    "node": {
     "id": "sidecar~10.128.2.23~back-golang-v1-<pod_id>.<user_namespace>~<user_namespace>.svc.cluster.local",
...

Finally, compare both files differences and draw a definitive conclusion:

diff back-golang-v1-<pod_id>-destination-rule-orig.json back-golang-v1-<pod_id>-destination-rule-custom.json

  • Visit the front, front-javascript-golang-<user_namespace>, route via your web browser. In addition, configure 1000 retries with 1 interval and push - JUMP- button and ensure the following message is displaying in your screen
